# Addoners Optimizer v1.5 - Production Upgrade

## ✅ Completed Features

### 1. **Cooldown System** (NEW)
- **File**: `CooldownSystem.java`
- **Purpose**: Prevents rapid optimization level switching
- **Cooldown Duration**: 3 seconds (configurable)
- **Features**:
  - `isCooldownExpired()` - checks if enough time has passed
  - `shouldApply(level)` - prevents re-applying same level
  - `getTimeRemainingMs()` - returns remaining cooldown time
  - Server-compatible design (no Client-only APIs)

### 2. **FPS Smoothing System** (ENHANCED)
- **File**: `FPSMonitor.java` (already implemented)
- **Approach**: Rolling average of last 10 FPS samples
- **Impact**: Eliminates optimization thrashing from momentary FPS spikes
- **API**:
  - `getFPS()` - returns smoothed average FPS
  - `getRawFPS()` - returns unsmoothed FPS
  - `reset()` - clears sample buffer

### 3. **Status Overlay (HUD)** (NEW)
- **File**: `StatusOverlay.java`
- **Purpose**: Real-time visual feedback on optimizer state
- **Location**: Top-left corner of screen
- **Displays**:
  - Current optimization level (color-coded)
  - Current FPS
  - Active profile name
  - Shader status (ON/OFF)
- **Features**:
  - Toggleable via config
  - Zero performance impact (uses Fabric HudRenderCallback)
  - Only renders when game is active (hidden in menus)
  - Singleton pattern with lazy initialization

### 4. **Shader Awareness System** (ENHANCED)
- **File**: `ShaderDetector.java` (enhanced)
- **Features**:
  - Detects Iris, Canvas, and custom shaders
  - Reflection-based Iris detection (no hard dependency)
  - Safe fallback if shader loader unavailable
  - Periodic refresh (~5 seconds)
  
- **File**: `ShaderOptimizer.java`
- **New Method**: `adjustLevel(OptimizationLevel)`
  - Reduces aggressiveness when shaders active
  - Prevents render distance cuts below safe minimum
  - Preserves visual quality for shader packs

### 5. **Optimization Level Engine** (ENHANCED)
- **File**: `OptimizationLevel.java`
- **Levels**:
  - **LOW**: FPS ≥ 80 (minimal intervention)
  - **MEDIUM**: FPS 50–79 (moderate adjustments)
  - **AGGRESSIVE**: FPS < 50 (maximum optimization)
- **Exposed Getters**:
  - `getDisplayName()` - human-readable level name
  - `getDescription()` - detailed explanation
  - `fromFps(int fps)` - determines level from FPS

### 6. **OptimizerEngine v1.5** (REBUILT)
- **File**: `OptimizerEngine.java` (completely refactored)
- **Architecture**:
  1. Sample FPS (rolling average)
  2. Resolve optimization level
  3. Detect shader changes (~every 5 seconds)
  4. Check cooldown window
  5. Only optimize if cooldown expired AND level changed
  6. Update status overlay
  7. Emit structured logs
  
- **New Methods**:
  - `getCurrentLevel()` - current effective level
  - `getCurrentFps()` - last sampled FPS
  - `isCooldownActive()` - cooldown status
  - `getCycleCount()` - total cycles executed

- **Structured Logging** (v1.5):
  ```
  [Addoners Optimizer] FPS: 62 → MEDIUM
  [Addoners Optimizer] Cooldown active: false
  [Addoners Optimizer] Profile applied: quartzglow
  [Addoners Optimizer] Shader detected: true
  ```

### 7. **Dynamic Optimizer** (ENHANCED)
- **File**: `DynamicOptimizer.java`
- **New Features**:
  - Shader-aware optimization path
  - Safe render distance capping with shaders
  - Intelligent setting layering:
    1. Profile base settings
    2. Dynamic rule overrides
    3. Shader safety layer
    4. Fallback defaults

### 8. **Config System v1.5** (ENHANCED)
- **File**: `ModConfig.java`
- **New Fields**:
  - `showStatus` (boolean) - enable/disable overlay
  - `shaderOptimization` (boolean) - enable/disable shader awareness
- **Maintained Fields**:
  - `enabled` - master switch
  - `debugLogs` - diagnostic logging
  - `optimizerIntervalTicks` - cycle frequency
  
- **Backward Compatibility**: Missing fields default to safe values during load

### 9. **Settings Menu** (COMPLETELY REBUILT)
- **File**: `OptimizerConfigScreen.java`
- **Integration**: Mod Menu (v1.5+ feature)
- **GUI Controls**:
  - Enable/Disable toggle (status-colored)
  - Debug Logs toggle (ON/OFF)
  - Show Status toggle (NEW - v1.5)
  - Shader Optimization toggle (NEW - v1.5)
  - Cycle Interval selector (1-200 ticks)
  - Save & Close button
  - Cancel button
  
- **Visual Design**:
  - Centered layout
  - Color-coded toggles (green=ON, red=OFF)
  - Real-time status display
  - ESC to cancel

### 10. **Mod Menu Integration** (NEW)
- **File**: `ModMenuIntegration.java`
- **Purpose**: Registers config screen with Mod Menu
- **Dependency**: Added to `build.gradle`
  ```gradle
  modImplementation "com.terraformersmc:modmenu:12.1.0+mc26.1"
  ```
- **Entry Point**: Registered in `fabric.mod.json`

###11. **Profile System** (.addoners - CORE)
- **Location**: `config/teamaddoners/profiles/`
- **Structure**:
  ```json
  {
    "type": "optimizer_profile",
    "name": "quartzglow",
    "settings": {
      "renderDistance": 10,
      "smoothLighting": true,
      "particles": 1
    },
    "dynamicRules": {
      "fpsBelow60": {
        "renderDistance": 8,
        "particles": 0
      },
      "fpsBelow30": {
        "renderDistance": 6,
        "particles": 0
      }
    }
  }
  ```
- **Priority Order** (v1.5):
  1. Profile settings (highest)
  2. Dynamic rules (FPS-based)
  3. Shader safety layer
  4. Engine fallback (lowest)

### 12. **Rule Engine** (v1.5)
- **File**: `RuleEngine.java`
- **Supported Rules**:
  - `fpsBelow60` - applied when FPS < 60
  - `fpsBelow30` - stacked when FPS < 30
  - `fpsBelow20` - highest priority when FPS < 20
  
- **Behavior**: Rules layer progressively, lower FPS = stronger overrides

### 13. **Addoners Shader Synergy** (USP - v1.5)
- **Profiles**: `default.addoners`, `quartzglow.addoners`, `skygleam.addoners`
- **Integration**:
  - Quartzglow profile → shader-friendly optimization
  - Skygleam profile → balanced optimization
  - Auto-detected shader matching
  
- **Settings Adjustments**:
  - Reduced render distance cuts with shaders
  - Preserved visual quality
  - Maintained performance targets

### 14. **Server-Side Future Compatibility** (ARCHITECTURE)
- **Design Principles**:
  - ✅ CooldownSystem - No client-only APIs
  - ✅ OptimizationLevel enum - Pure logic
  - ✅ RuleEngine - Pure JSON/config
  - ✅ ModConfig - Serializable
  - ⚠️ StatusOverlay - Client-only (as expected)
  - ⚠️ Rendering - Fabric client APIs (as expected)
  
- **Future Expansion Ready**:
  - Core logic can run on client + server
  - Profile system is modular
  - Settings are JSON-agnostic
  - No tight coupling to client-only rendering
